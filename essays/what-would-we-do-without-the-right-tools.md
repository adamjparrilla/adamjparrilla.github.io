## Why are tools important?

The development and use of tools may be what sets apart our species from others. Through tools, we have been able to accomplish tasks that the human body would be unable to do alone. Having the right tool in a situation can be the difference between an impossible job and a simple one. At this point, any device or implement meant to carry out a function can be considered a tool. Whether that be physical equipment like a hammer or one of the millions of applications online tools are a part of our everyday lives and make our lives easier.

##  How does this relate at all to Software Engineering?

Tools can be seen everywhere when developing software, such as GitHub, IntelliJ, or even DevTools just to name a few. In basically everything, the use of tools can improve how we do things. Therefore, it should be no surprise that without tools we would be hindered greatly in what we can accomplish in software engineering. As stated earlier, the right development tool can mean the difference between spending days or minutes when developing software. With that in mind, it’s clear to see why tools are such an important thing to consider when working in Software Engineering.

## How have Development Environments helped me?

 One of the most crucial parts of any software development starts at the beginning. What development environment do you choose? And what is a Development Environment in the first place? Well, it is an application or software tool used by programmers to create programs. Making writing code much more productive and providing faster set up along with various tools to make it easier. So, if the right tool can make or break anything, deciding on the right environment to develop your code in is just as important. Personally, as I continue to learn more about Computer Science, IDE’s (integrated development environment) will also continue to be a part of all my future projects. Having the right tool for the job is important, so having a productive environment to work in will be crucial for getting all my future projects done well.

## What makes Design Patterns a useful tool for any Software Engineer?

Design patterns are in general a reusable solution to a commonly occurring problem within software design. While it may not be a complete or finished design, it gives a programmer format to follow, so you’re not starting from scratch. Similar to scheduling our daily lives or teacher in a school lesson, we use design patterns for structure, consistency, and flexibility. With the help of design patterns as a point to go from, it makes finding solutions much faster and more reliable. In my experience, using design patterns can act as a baseline to be referred to when you’re making something similar. For example, in my ICS 314 course instead of spending hours creating our final web design from scratch. My group used a design pattern like what we wanted and grew on it from there. This made our code much easier to follow and change because our design pattern tool was already proven to work before.

## Why are Coding Standards such an important tool?

While it may not seem like such a necessary tool, coding standards are key to ensuring your code’s safety, reliability, and security. One benefit that I have personally found when using coding standards right away, is cleaner and more standardized code. This means less code litter of things like large areas of commented-out code, poor variable names, and more. Coding standards also mean that it’s easier to maintain my code because even as a database continues to grow, my code is always following a set standard. Another thing that I’ve found is when working in a group, coding standards are an important tool, so everyone can understand my code. With more consistency among a group, working together becomes easier and you can spend less time trying to understand what a group member has added.

## So why were tools important?

Finally, tools help us to accomplish tasks and goals that we would be unable to complete otherwise. Tools like, Development Environments, Design Patterns, and Coding Standards are just a few of the things software engineers can use when creating various kinds of software. So, now that you know about these tools, how will you use them?
